# Observability

Validation skills focused on making systems debuggable in production.

Use these skills to verify that your system produces enough signal to understand what is happening — and to diagnose issues quickly when something goes wrong.

## Skills

| File | Description |
|------|-------------|
| [logging-audit.md](./logging-audit.md) | Evaluate whether the system produces meaningful, structured, and actionable logs. |
| [debug-readiness.md](./debug-readiness.md) | Assess whether a production bug can be investigated and reproduced with the current level of visibility. |

## When to use

Run an observability skill when you:

- Add new features or critical user flows
- Modify error handling or exception propagation
- Integrate with external services
- Prepare a feature for production release
