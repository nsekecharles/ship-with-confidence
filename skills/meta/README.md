# Meta

Skills focused on understanding and taking ownership of the code you ship.

Use these skills when you are not fully confident you understand what you are about to push — especially when working with AI-generated code, copied patterns, or unfamiliar libraries.

## Skills

| File | Description |
|------|-------------|
| [explain-unknowns.md](./explain-unknowns.md) | Surface hidden assumptions, implicit logic, and blind spots in code you did not fully write yourself. |

## When to use

Run a meta skill when you:

- Used AI to generate or refactor a significant portion of code
- Copied a pattern or snippet you do not fully understand
- Are about to ship code that "works" but feels opaque
- Need to explain the code to a teammate or reviewer

## Why it matters

The biggest risk of AI-assisted development is not broken code.
It is shipping code you cannot explain or debug when something goes wrong.
