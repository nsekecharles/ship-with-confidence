# 🕵️ Attack Simulation

## 🎯 Objective

Think like an attacker to identify vulnerabilities before they are exploited.

---

## 🧠 Prompt to run

You are an attacker.

Your goal is to exploit this system.

Identify:

- entry points
- attack vectors
- realistic exploitation scenarios

Rank vulnerabilities by severity and propose fixes.

---

## ⚠️ What to look for

### Frontend
- XSS possibilities
- Unsafe rendering of user input

### Backend
- Injection risks (SQL, NoSQL, command)
- Missing input validation

### Mobile
- Reverse engineering risks
- Hardcoded secrets
- Insecure storage

---

## ✅ Expected outcome

- Identified attack scenarios  
- Prioritized vulnerabilities  
- Actionable fixes  

---

## 💡 Why it matters

Security is not about defense.

👉 It’s about anticipating attacks.
