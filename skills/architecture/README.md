# Architecture

Validation skills focused on maintainability and system design.

Use these skills to evaluate the structural quality of your code — catching hidden coupling, scalability limits, and design issues before they become expensive to fix.

## Skills

| File | Description |
|------|-------------|
| [coupling-detector.md](./coupling-detector.md) | Identify tightly coupled components, hidden dependencies, and unclear separation of concerns. |
| [scalability-check.md](./scalability-check.md) | Evaluate whether the current architecture can handle growth in users, data, and features. |

## When to use

Run an architecture skill when you:

- Add new modules or services
- Introduce shared logic or cross-cutting concerns
- Refactor existing components
- Design or change data flow between layers
